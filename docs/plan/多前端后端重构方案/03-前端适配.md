## 2.2.1 ASTToIRConverter — AST→IR 转换层

这是 C++ 前端的核心新组件，将现有 CodeGen 逻辑从"直接生成 LLVM IR"改为"生成 BTIR"：

### 完整类定义

```cpp
class ASTToIRConverter {
  ir::IRContext& IRCtx;              /// IR 内存管理上下文（新增）
  ir::IRTypeContext& TypeCtx;
  const TargetLayout& Layout;
  DiagnosticsEngine& Diags;
  unique_ptr<ir::IRModule> TheModule;
  unique_ptr<ir::IRBuilder> Builder;

  // 类型映射
  unique_ptr<IRTypeMapper> TypeMapper;

  // 值映射：Decl → ir::IRValue*
  DenseMap<const Decl*, ir::IRValue*> DeclValues;
  // 全局变量映射
  DenseMap<const VarDecl*, ir::IRGlobalVariable*> GlobalVars;
  // 函数映射
  DenseMap<const FunctionDecl*, ir::IRFunction*> Functions;
  // 局部变量映射
  DenseMap<const VarDecl*, ir::IRValue*> LocalDecls;

  // 子组件（从现有 CodeGen 模块重构而来）
  unique_ptr<IREmitExpr> ExprEmitter;       // 从 CodeGenExpr 重构
  unique_ptr<IREmitStmt> StmtEmitter;       // 从 CodeGenStmt 重构
  unique_ptr<IREmitCXX> CXXEmitter;         // 从 CGCXX 重构
  unique_ptr<IRConstantEmitter> ConstEmitter; // 从 CodeGenConstant 重构

public:
  /// 构造函数：所有参数必须非空，IRCtx由CompilerInstance持有
  /// 约束：IRCtx必须已初始化（IRContext::IRContext()），Layout必须与目标平台一致
  /// 约束：Diags必须已绑定SourceManager
  ASTToIRConverter(ir::IRContext& IRCtx,
                   ir::IRTypeContext& Ctx,
                   const TargetLayout& Layout,
                   DiagnosticsEngine& Diags);

  /// 主转换入口：返回IRConversionResult（参考02 §2.1.13）
  /// 错误时返回IRConversionResult::getInvalid()，成功时返回包含IRModule的结果
  /// 约束：TU必须非空，TU->isInvalid()==false时才执行转换
  ir::IRConversionResult convert(TranslationUnitDecl* TU);

  // 类型映射接口
  /// 将QualType映射为IRType，失败时返回emitErrorType()并发出diag::err_ir_type_mapping_failed
  ir::IRType* mapType(QualType T);

  // 函数/变量生成
  /// 生成IRFunction，失败时生成空函数体（仅ret void）并发出diag::err_ir_function_conversion_failed
  ir::IRFunction* emitFunction(FunctionDecl* FD);
  /// 生成IRGlobalVariable，失败时用IRConstantUndef作为初始值
  ir::IRGlobalVariable* emitGlobalVar(VarDecl* VD);

  // C++ 特有
  void convertCXXConstructor(CXXConstructorDecl* Ctor, ir::IRFunction* IRFn);
  void convertCXXDestructor(CXXDestructorDecl* Dtor, ir::IRFunction* IRFn);
  ir::IRValue* convertCXXConstructExpr(CXXConstructExpr* CCE);
  ir::IRValue* convertVirtualCall(CXXMemberCallExpr* MCE);

  // 错误恢复（新增）
  ir::IRValue* emitErrorPlaceholder(ir::IRType *T);  /// 发出错误占位值（IRConstantUndef::get(T)）
  ir::IRType* emitErrorType();                         /// 发出错误类型占位（IROpaqueType("error")）
};
```

  ///
  /// | QualType Kind              | IRType                     | 映射规则                                     |

  /// | BuiltinType::Bool           | IRIntegerType(1)           | TypeCtx.getBoolType()                        |
  /// | BuiltinType::Char_U/S       | IRIntegerType(8)           | TypeCtx.getInt8Ty()                          |
  /// | BuiltinType::UChar/SChar    | IRIntegerType(8)           | TypeCtx.getInt8Ty()                          |
  /// | BuiltinType::Short/UShort   | IRIntegerType(16)          | TypeCtx.getIntType(16)                       |
  /// | BuiltinType::Int/UInt       | IRIntegerType(32)          | TypeCtx.getInt32Ty()                         |
  /// | BuiltinType::Long/ULong     | IRIntegerType(Layout.LongSize*8) | 平台相关：x86_64=64, x86_32=32, aarch64=64 |
  /// | BuiltinType::LongLong/ULongLong | IRIntegerType(64)     | TypeCtx.getInt64Ty()                         |
  /// | BuiltinType::Float          | IRFloatType(32)            | TypeCtx.getFloatTy()                         |
  /// | BuiltinType::Double         | IRFloatType(64)            | TypeCtx.getDoubleTy()                        |
  /// | BuiltinType::LongDouble     | IRFloatType(Layout.LongDoubleSize*8) | 平台相关：x86_64=80, aarch64=64 |
  /// | PointerType                 | IRPointerType(Pointee)     | mapType(Pointee) → getPointerType()          |
  /// | ReferenceType               | IRPointerType(Pointee)     | 引用→指针（语义等价）                        |
  /// | ArrayType(Constant)         | IRArrayType(Element, N)    | mapType(Element) → getArrayType()            |
  /// | RecordType(Struct/Class)    | IRStructType(Name, Fields) | 逐字段mapType → getStructType()              |
  /// | FunctionProtoType           | IRFunctionType(Ret, Params)| mapType(Ret) + mapTypes(Params)              |
  /// | EnumType                    | IRIntegerType(BitWidth)    | BitWidth = Layout.getTypeSizeInBits(EnumDecl)|
  /// | TypedefType                 | 递归mapType(UnderlyingType)| 透明映射                                     |
  /// | TemplateSpecializationType  | 递归mapType(Desugar)       | 透明映射                                     |
  /// | DependentType               | IROpaqueType(Name)         | 未决类型→不透明类型占位                      |
  /// | VectorType                  | IRVectorType(N, Element)   | mapType(Element) → getVectorType()           |
  /// | ComplexType                 | IRStructType("_complex", [T,T]) | 复数→结构体                            |
  /// | BlockType                   | IRPointerType(IRFunctionType(...)) | Block→函数指针                         |
  /// | ObjCObjectPointerType        | IRPointerType(IRIntegerType(8)) | ObjC→不透明指针（远期）               |
};

## 2.2.2 现有 CodeGen 重构策略

### 三阶段渐进式（取自方案1）

| 阶段 | 策略 | 说明 |
|------|------|------|
| **共存期** | 新增ASTToIRConverter，与CodeGenModule并行 | CppFrontend用新路径，旧路径不变 |
| **迁移期** | 逐步将CodeGenFunction逻辑迁移到ASTToIRConverter | 每迁移一个Expr类型，验证BTIR正确性 |
| **替换期** | ASTToIRConverter覆盖全部后，CodeGenModule仅LLVM后端内部使用 | CodeGen从"前端组件"变为"后端内部实现" |

### 重构映射表（取自方案0）

| 现有类 | 重构后 | 变化 |
|--------|--------|------|
| `CodeGenModule` | `ASTToIRConverter` | `llvm::Module` → `ir::IRModule`；`llvm::LLVMContext` → `ir::IRTypeContext` |
| `CodeGenFunction` | `IREmitFunction` | `IRBuilderTy` → `ir::IRBuilder`；`llvm::Function*` → `ir::IRFunction*`；`llvm::Value*` → `ir::IRValue*` |
| `CodeGenTypes` | `IRTypeMapper` | `QualType → llvm::Type*` → `QualType → ir::IRType*` |
| `CodeGenConstant` | `IRConstantEmitter` | `llvm::Constant*` → `ir::IRConstant*` |
| `CodeGenExpr`(99KB) | `IREmitExpr` | 所有 `Emit*Expr` 返回 `ir::IRValue*` |
| `CodeGenStmt`(50KB) | `IREmitStmt` | 所有 `Emit*Stmt` 使用 `ir::IRBuilder` |
| `CGCXX`(81KB) | `IREmitCXX` | vtable/RTTI/继承使用 `ir::IRGlobalVariable`/`ir::IRFunction` |
| `CGDebugInfo` | `IRDebugInfo` | `llvm::DIType*` → `ir::DebugMetadata` |
| `Mangler` | `IRMangler` | 依赖 `ASTContext&`+`TargetLayout&`，不依赖 `CodeGenModule` |

### 关键替换模式

```cpp
// 旧：直接生成 LLVM IR
llvm::Value *V = Builder.CreateAdd(LHS, RHS, "addtmp");

// 新：生成 BTIR
ir::IRValue *V = Builder.createAdd(LHS, RHS, "addtmp");

// 旧：使用 llvm::Type*
llvm::Type *Int32Ty = llvm::Type::getInt32Ty(LLVMCtx);

// 新：使用 ir::IRType*
ir::IRIntegerType *Int32Ty = TypeCtx.getInt32Ty();

// 旧：创建 llvm::Function
llvm::Function *F = llvm::Function::Create(FTy, Linkage, Name, TheModule);

// 新：创建 ir::IRFunction
ir::IRFunction *F = IRModule->getOrInsertFunction(Name, FTy);
```

### 精确替换模式表

| 旧API签名 | 新API签名 | 自动替换 | 备注 |
|-----------|-----------|---------|------|
| `llvm::Value* Builder.CreateAdd(llvm::Value*, llvm::Value*, const Twine&)` | `ir::IRValue* Builder.createAdd(ir::IRValue*, ir::IRValue*, StringRef)` | ✓ | 返回类型和参数类型替换 |
| `llvm::Value* Builder.CreateSub(...)` | `ir::IRValue* Builder.createSub(...)` | ✓ | 同上 |
| `llvm::Value* Builder.CreateLoad(llvm::Type*, llvm::Value*, const Twine&)` | `ir::IRValue* Builder.createLoad(ir::IRType*, ir::IRValue*, StringRef)` | ✓ | 参数顺序不变 |
| `llvm::Value* Builder.CreateStore(llvm::Value*, llvm::Value*)` | `ir::IRValue* Builder.createStore(ir::IRValue*, ir::IRValue*)` | ✓ | 参数顺序不变 |
| `llvm::Value* Builder.CreateCall(llvm::FunctionCallee, ArrayRef<llvm::Value*>)` | `ir::IRValue* Builder.createCall(ir::IRFunction*, ArrayRef<ir::IRValue*>)` | ✓ | FunctionCallee→IRFunction* |
| `llvm::Value* Builder.CreateGEP(llvm::Type*, llvm::Value*, ArrayRef<llvm::Value*>)` | `ir::IRValue* Builder.createGEP(ir::IRType*, ir::IRValue*, ArrayRef<ir::IRValue*>)` | ✓ | 需显式传入元素类型 |
| `llvm::Type* CodeGenTypes::ConvertType(QualType)` | `ir::IRType* IRTypeMapper::mapType(QualType)` | ✓ | 类名+方法名替换 |
| `llvm::Constant* CodeGenConstant::EmitConst(...)` | `ir::IRConstant* IRConstantEmitter::EmitConst*(...)` | ✓ | 方法名细化 |
| `llvm::Function* CodeGenModule::GetAddrOfFunction(GlobalDecl)` | `ir::IRFunction* ASTToIRConverter::emitFunction(FunctionDecl*)` | ✗ | 语义不同：旧版返回地址，新版生成IR函数 |
| `llvm::GlobalVariable* CodeGenModule::CreateOrInitGlobalVar(...)` | `ir::IRGlobalVariable* ASTToIRConverter::emitGlobalVar(VarDecl*)` | ✗ | 初始化逻辑不同：旧版直接生成LLVM常量，新版生成IR常量 |
| `Builder.CreateInvoke(Callee, NormalBB, UnwindBB, Args)` | `Builder.createInvoke(IRFn, NormalBB, UnwindBB, Args)` | ✗ | 异常语义需DialectLoweringPass处理（invoke→call+landingpad） |
| `CGCXX.EmitVTable(RD)` | `IREmitCXX::EmitVTable(RD)` | ✗ | VTable生成逻辑从全局常量变为IR占位+后端填充 |
| `CGCXX.EmitRTTI(RD)` | `IREmitCXX::EmitRTTI(RD)` | ✗ | 同VTable |
| `Builder.CreateLandingPad(...)` | 无直接对应 | ✗ | 需DialectLoweringPass降级为bt_core指令 |

### 不可自动替换的模式（需手动调整）

1. **异常处理**：`CreateInvoke`/`CreateLandingPad`/`CreateResume` → 需在bt_cpp Dialect中定义，由DialectLoweringPass降级
2. **VTable/RTTI生成**：`CGCXX.EmitVTable()` → IR层仅创建占位全局变量（IROpaqueType），LLVM后端填充具体布局
3. **dynamic_cast**：`CGCXX.EmitDynamicCast()` → 降级为`__dynamic_cast`函数调用（bt_cpp Dialect）
4. **DebugInfo**：`CGDebugInfo` → IR调试元数据（ir::debug::IRDebugMetadata），映射方式完全不同
5. **ABI特定代码**：`ABIInfo::classifyArgumentType()` → 后端层处理，前端不涉及

| IREmitCXXInherit.cpp | `IREmitCXXInherit.h`, `IREmitCXXLayout.h`, `IRBuilder.h` | 继承需要偏移计算 |
| IREmitCXXThunk.cpp | `IREmitCXXThunk.h`, `IREmitCXXVTable.h`, `IRBuilder.h`, `IRMangler.h` | Thunk需要VTable和名称修饰 |

**注意**：所有子文件都不再 `#include "CodeGenModule.h"`，改为 `#include "ASTToIRConverter.h"`。

**适配要点**：
- 所有 `llvm::GlobalVariable*` → `ir::IRGlobalVariable*`
- 所有 `llvm::Function*` → `ir::IRFunction*`
- 所有 `llvm::Value*` → `ir::IRValue*`
- 所有 `IRBuilderTy` 操作 → `ir::IRBuilder` 操作
- vtable 布局逻辑不变（它操作的是偏移量计算，与 IR 无关）
- RTTI 结构不变（它生成的是全局常量，只需改类型）

**VTable/RTTI 归属**（取自方案1策略）：
- 布局计算 → 移入 ASTToIRConverter，输出 IR 结构体类型
- 构造/析构 → 移入 ASTToIRConverter，输出 IR 指令序列
- **VTable/RTTI → 移入 LLVM 后端的 IRToLLVMConverter**（Itanium ABI 特定概念）
- Mangler → 独立为 IRMangler

## 2.2.4 Mangler 的后端无关化

当前 `Mangler` 持有 `CodeGenModule&` 引用，仅用于获取 `ASTContext` 和 `TargetInfo`。

**重构方案**（取自方案1）：

- **去掉对 `CodeGenModule` 的依赖**：改为依赖 `ASTContext` 和 `TargetLayout`
- **Mangler 不需要修改核心逻辑**：它只生成字符串，不涉及 IR 类型
- **签名变更**：`Mangler(CodeGenModule &)` → `IRMangler(ASTContext &, const TargetLayout &)`

理由：`TargetLayout` 属于 IR 层，不依赖 LLVM，更符合 R6。

## 2.2.5 TargetInfo/ABIInfo 的归属调整

**调整方案**（取自方案1）：

| 组件 | 当前归属 | 重构后归属 | 说明 |
|------|---------|-----------|------|
| TargetInfo | CodeGen/ | IR/(TargetLayout) + Backend/ | 布局信息→IR层，LLVM特定→后端 |
| ABIInfo | CodeGen/ | Backend/ | ABI分类是后端特定概念 |
| ABIArgInfo | CodeGen/CodeGenTypes.h | IR/ | 参数传递方式是IR层概念 |

**ABIArgInfo 改造**（方案1独有）：
将 `ABIArgInfo` 中的 `llvm::Type* SRetType` 替换为 `ir::IRType* SRetType`。

**目录调整**：

```
调整前：include/blocktype/CodeGen/TargetInfo.h
调整后：include/blocktype/Target/TargetInfo.h（布局信息委托给IR层TargetLayout）

调整前：include/blocktype/CodeGen/ABIInfo.h
调整后：include/blocktype/Backend/ABIInfo.h（ABI是后端概念）
```

理由：
- `TargetInfo` 提供类型大小/对齐信息，前端和后端都需要
- `ABIInfo` 提供 ABI 分类，是后端特定概念
- `TargetLayout` 是 IR 层的布局信息镜像，不依赖 LLVM

## 2.2.6 AST→IR 错误恢复策略

ASTToIRConverter 在转换过程中遇到错误时，采用**占位+继续**策略，尽可能多地收集错误信息：

| 错误场景 | 恢复策略 | 占位值 | DiagID枚举值 | 占位值创建方式 |
|---------|---------|--------|-------------|--------------|
| QualType 无法映射到 IRType | 发出 Diag，用 `IROpaqueType` 占位 | `IROpaqueType("error")` | `diag::err_ir_type_mapping_failed` | `ir::IROpaqueType::get(TypeCtx, "error")` |
| 表达式转换失败 | 发出 Diag，用 `IRConstantUndef` 占位 | `IRConstantUndef::get(T)` | `diag::err_ir_expr_conversion_failed` | `ir::IRConstantUndef::get(TypeCtx, T)` |
| 语句转换失败 | 发出 Diag，跳过该语句 | — | `diag::err_ir_stmt_conversion_failed` | 无占位值，直接跳过 |
| 函数体转换失败 | 发出 Diag，生成空函数体（仅 ret void） | `createRetVoid()` | `diag::err_ir_function_conversion_failed` | `Builder.createRetVoid()` |
| 全局变量初始化失败 | 发出 Diag，用 `IRConstantUndef` 作为初始值 | `IRConstantUndef::get(T)` | `diag::err_ir_globalvar_init_failed` | `ir::IRConstantUndef::get(TypeCtx, T)` |
| C++特有语义转换失败 | 发出 Diag，用 `IRConstantUndef` 占位 | `IRConstantUndef::get(T)` | `diag::err_ir_cxx_semantic_failed` | `ir::IRConstantUndef::get(TypeCtx, T)` |
| VTable生成失败 | 发出 Diag，用空全局变量占位 | `IRGlobalVariable(Name, IROpaqueType)` | `diag::err_ir_vtable_generation_failed` | `IRModule->getOrInsertGlobal(Name, IROpaqueType)` |
| RTTI生成失败 | 发出 Diag，用空全局变量占位 | `IRGlobalVariable(Name, IROpaqueType)` | `diag::err_ir_rtti_generation_failed` | `IRModule->getOrInsertGlobal(Name, IROpaqueType)` |

### DiagID 枚举值完整列表

```cpp
// 在 DiagnosticIDs.h 中新增（IR转换专用）
namespace diag {
  // IR类型映射错误
  DIAG(err_ir_type_mapping_failed,     Error, "cannot map QualType to IRType: %0")
  // IR表达式转换错误
  DIAG(err_ir_expr_conversion_failed,  Error, "failed to convert expression to IR: %0")
  // IR语句转换错误
  DIAG(err_ir_stmt_conversion_failed,  Error, "failed to convert statement to IR: %0")
  // IR函数转换错误
  DIAG(err_ir_function_conversion_failed, Error, "failed to convert function body to IR: %0")
  // IR全局变量初始化错误
  DIAG(err_ir_globalvar_init_failed,   Error, "failed to initialize global variable in IR: %0")
  // IR C++特有语义错误
  DIAG(err_ir_cxx_semantic_failed,     Error, "failed to convert C++ semantic to IR: %0")
  // IR VTable生成错误
  DIAG(err_ir_vtable_generation_failed, Error, "failed to generate VTable in IR: %0")
  // IR RTTI生成错误
  DIAG(err_ir_rtti_generation_failed,  Error, "failed to generate RTTI in IR: %0")
  // IR警告（非错误，不阻止编译）
  DIAG(warn_ir_type_demotion,          Warning, "IR type demotion may lose precision: %0 to %1")
  DIAG(warn_ir_unsupported_feature,    Warning, "IR feature not supported by target backend: %0")
}
```

**与 DiagnosticsEngine 的集成**：
- 所有 IR 转换错误通过 `DiagnosticsEngine` 报告，复用现有的 SourceLocation 体系
- 新增 IR 专用 DiagID（如 `diag::err_ir_type_mapping_failed`、`diag::err_ir_expr_conversion_failed`）
- IR 转换错误不受 SFINAE 抑制（硬错误）

**最终结果**：`convert()` 返回 `IRConversionResult`，调用者通过 `isInvalid()` 判断是否有错误，通过 `getNumErrors()` 获取错误数量。

## 2.2.7 前端-IR 契约验证

前端 `compile()` 产出的 IRModule 必须满足以下契约不变量：

| 契约 | 说明 | 验证函数 |
|------|------|---------|
| C1: VerifierPass 通过 | IRModule 通过 VerifierPass 验证 | `contract::verifyIRModuleContract()` |
| C2: 类型完整性 | 所有 IRType 非 Opaque | `contract::verifyTypeCompleteness()` |
| C3: 函数非空 | 每个 IRFunction 至少有一个 IRBasicBlock | `contract::verifyFunctionNonEmpty()` |
| C4: 终结指令 | 每个 IRBasicBlock 恰好一个终结指令 | `contract::verifyTerminatorContract()` |
| C5: 类型一致性 | 所有 IRValue 的类型与使用点类型一致 | `contract::verifyTypeConsistency()` |
| C6: TargetTriple 有效 | IRModule 的 TargetTriple 非空 | `contract::verifyTargetTripleValid()` |

**在管线中的位置**：

```
Frontend::compile() → IRConversionResult
    ↓ (isUsable())
contract::verifyAllContracts(IRModule)  ← 新增：契约验证
    ↓ (通过)
IRPipeline::run() → VerifierPass + OptimizationPasses
    ↓
Backend->getCapability() → 能力检查
    ↓
Backend::emitObject()
```

`--verify-btir` 选项触发契约验证。默认在 Debug 构建中启用，Release 构建中跳过。

## 2.2.8 编译器插件系统 — 前端插件

> 来源：09-行业前沿补充优化方案 §2.6

借鉴 LLVM New Pass Manager + Clang Plugin 的三层插件架构，为前端引入插件扩展能力。前端插件允许第三方在编译过程中注入自定义行为，如自定义诊断、AST 遍历回调等。

### 插件类型枚举

```cpp
namespace blocktype::plugin {

/// 插件类型枚举
enum class PluginType : uint8_t {
  IRPass,          /// IR Pass插件
  Frontend,        /// 前端插件
  Backend,         /// 后端插件
  Diagnostic,      /// 诊断插件（自定义警告/错误处理）
  Analysis,        /// 分析插件（代码分析/静态检查）
};

/// 插件元数据
struct PluginInfo {
  std::string Name;
  std::string Version;
  std::string Description;
  PluginType Type;
  std::vector<std::string> ProvidedPasses;   /// 提供的Pass名称
  std::vector<std::string> RequiredDialects; /// 需要的Dialect
};

/// 插件基类
class CompilerPlugin {
public:
  virtual ~CompilerPlugin() = default;
  virtual PluginInfo getInfo() const = 0;
  virtual bool initialize(CompilerInstance& CI) = 0;
  virtual void finalize() = 0;
};

/// 插件管理器
class PluginManager {
  DenseMap<StringRef, std::unique_ptr<CompilerPlugin>> LoadedPlugins;
  std::vector<void*> LoadedLibraries;  /// dlopen句柄

public:
  /// 从动态库加载插件
  bool loadPlugin(StringRef Path);

  /// 卸载插件
  bool unloadPlugin(StringRef Name);

  /// 获取已加载插件
  CompilerPlugin* getPlugin(StringRef Name) const;

  /// 注册插件提供的IR Pass
  void registerPluginPasses(CompilerInstance& CI);

  /// 列出所有已加载插件
  void listPlugins(raw_ostream& OS) const;
};

} // namespace blocktype::plugin
```

### 前端插件与 FrontendRegistry 的关系

```
FrontendRegistry（现有）
├── CppFrontend       /// C++前端（内置）
├── CFrontend         /// C前端（未来）
└── ...               /// 其他内置前端

PluginManager（新增，上层管理器）
├── FrontendRegistry 不变，插件系统是上层管理器
├── 前端插件可通过 PluginManager 注册自定义诊断/分析回调
└── 前端插件不替换 FrontendRegistry 中的前端，而是扩展前端行为
```

**兼容性说明**：
- `FrontendRegistry` 保持不变，插件系统是上层管理器
- Phase A-E 不实现插件系统，仅预留 `PluginManager` 接口
- Phase F 实现基础插件加载
- Phase G 实现IR Pass插件注册

### 编译选项

```
--fplugin=<path>                加载编译器插件
--fplugin-arg-<name>-<arg>      传递参数给插件
--fplugin-list                  列出已加载插件
```

## 2.2.9 跨语言互操作 — FFI前端适配

> 来源：09-行业前沿补充优化方案 §2.11

在IR层设计FFI（Foreign Function Interface）接口，支持C++前端与其他语言互操作。前端适配层负责将外部函数声明转换为IR层的FFI声明。

### FFI IR层定义（引用02）

> **定义引用**：`CallingConvention`枚举、`FFIFunctionDecl`类、`FFITypeMapper`类的完整定义见 [02-IR层实现.md](02-IR层实现.md) §2.1.24。本节仅描述前端适配层的集成逻辑。

### C语言零开销映射规则（引用02）

> **映射规则表**：C类型→IR类型的完整映射见 [02-IR层实现.md](02-IR层实现.md) §2.1.24。C语言为零开销映射（类型直接对应），其他语言通过C ABI桥接。

### FFI与 ASTToIRConverter 的集成

```
ASTToIRConverter::convert()
    ↓ 遇到外部函数声明（如 extern "C" int foo(int)）
FFIFunctionDecl::createIRDeclaration()
    ↓ 生成IR层 declare 指令（带调用约定属性）
ASTToIRConverter 继续处理函数体
    ↓ 遇到外部函数调用
FFIFunctionDecl::createCallWrapper()
    ↓ 生成调用包装器（处理调用约定差异）
ASTToIRConverter 生成 call 指令
```

**集成要点**：
- `ASTToIRConverter` 在处理 `extern` 声明时，委托给 `FFIFunctionDecl` 生成IR声明
- C语言映射为零开销（类型直接对应），其他语言通过C ABI桥接
- Phase C 实现 `FFIFunctionDecl` 基础和 C语言映射
- Phase E 实现 `FFICoerce`/`FFIUnwind` 指令

## 2.2.10 编译器诊断升级 — 前端诊断增强

> 来源：09-行业前沿补充优化方案 §2.7

升级前端诊断系统为行业领先水平，引入结构化诊断输出、FixIt Hint（自动修复建议）、诊断交叉引用和分组管理。

### 结构化诊断信息

```cpp
namespace blocktype::diag {

/// 结构化诊断信息（供IDE/工具消费）
struct StructuredDiagnostic {
  /// 基础信息
  diag::Level Level;              /// 错误/警告/备注/修复提示
  std::string Message;            /// 诊断消息
  SourceLocation PrimaryLoc;      /// 主位置
  SmallVector<SourceRange, 4> Ranges;  /// 高亮范围

  /// 附加信息
  SmallVector<SourceRange, 2> RelatedLocs;  /// 相关位置（如声明点）
  std::string Category;           /// 诊断分类（如 "type-error", "semantic-warning"）
  std::string FlagName;           /// 控制标志名（如 "-Wuninitialized"）

  /// FixIt Hint（自动修复建议）
  struct FixItHint {
    SourceRange Range;            /// 要替换的范围
    std::string Replacement;      /// 替换文本
    std::string Description;      /// 修复描述
  };
  SmallVector<FixItHint, 2> FixIts;

  /// 诊断链（因果关联）
  struct DiagnosticNote {
    SourceLocation Loc;
    std::string Message;
  };
  SmallVector<DiagnosticNote, 4> Notes;

  /// IR层扩展（IR诊断专用）
  Optional<ir::DialectID> IRRelatedDialect;  /// 关联的IR Dialect
  Optional<ir::Opcode> IRRelatedOpcode;       /// 关联的IR操作码
};

} // namespace blocktype::diag
```

## 2.2.11 前端模糊测试

> 来源：09-行业前沿补充优化方案 §2.10

引入前端模糊测试器，通过生成随机合法C++子集代码来检测前端崩溃和断言失败，提升编译器鲁棒性。

### 前端模糊测试器

```cpp
namespace blocktype::testing {

/// 前端模糊测试器：生成随机源代码，检测前端崩溃
class FrontendFuzzer {
public:
  /// 生成随机合法C++子集代码
  /// 策略：从预定义的语法模板中随机组合，确保生成的代码语法合法
  /// - 随机选择声明类型（函数/变量/类/模板）
  /// - 随机选择表达式类型（算术/比较/调用/成员访问）
  /// - 随机选择语句类型（if/for/while/return/表达式语句）
  std::string generateRandomSource(uint64_t Seed);

  /// 从种子代码变异
  /// 变异策略：
  /// - 随机插入/删除/替换声明
  /// - 随机修改表达式操作符
  /// - 随机修改类型
  /// - 随机嵌套/展平控制流
  std::string mutateSource(StringRef Seed, uint64_t Seed);

  /// 模糊测试结果
  struct FrontendFuzzResult {
    bool Crashed;               /// 前端崩溃
    bool AssertionFailed;       /// 断言失败
    bool ProducedInvalidAST;    /// 生成了无效AST
    std::string TriggerSource;  /// 触发问题的源代码
    std::string ErrorOutput;    /// 错误输出
    uint64_t IterationsRun;     /// 已运行迭代数
  };

  /// 执行前端模糊测试
  FrontendFuzzResult fuzz(uint64_t MaxIterations, uint64_t Seed);

  /// 从种子源代码开始变异测试
  FrontendFuzzResult fuzzFromSeed(StringRef SeedSource,
                                   uint64_t MaxIterations);

  /// 最小化触发用例（delta debugging）
  std::string minimizeTrigger(StringRef TriggerSource);
};

} // namespace blocktype::testing
```

### 前端模糊测试管线

```
FrontendFuzzer::fuzz()
    ↓ 循环 MaxIterations 次
    ├── generateRandomSource(Seed) 或 mutateSource(Seed, Seed)
    ├── CppFrontend::compile(Source)
    ├── 检查：崩溃? 断言失败? AST无效?
    ├── 如发现问题 → minimizeTrigger() 缩小用例
    └── 报告触发用例
```

**实施要点**：
- Phase C 实现 `FrontendFuzzer` 基础框架
- 生成策略优先覆盖C++核心语法子集（类/模板/继承/异常）
- 与现有 lit 测试框架集成，模糊测试发现的问题自动生成测试用例
- 不阻塞主线开发，发现的bug按优先级处理

### 编译选项（测试专用）

```
--ftest-frontend-fuzz=<iterations>  运行前端模糊测试
--ftest-frontend-fuzz-seed=<seed>   模糊测试种子
```
