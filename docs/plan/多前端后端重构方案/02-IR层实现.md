## 2.1.3 IRType 详细设计

### 2.1.3.1 IRType 基类

```cpp
namespace blocktype::ir {

class IRType {
public:
  enum Kind : uint8_t {
    Void = 0, Bool = 1, Integer = 2, Float = 3,
    Pointer = 4, Array = 5, Struct = 6, Function = 7,
    Vector = 8, Opaque = 9
  };

  Kind getKind() const { return KindVal; }
  DialectID getDialect() const { return DialectVal; }

  /// 结构等价性检查：比较类型结构而非指针。
  /// 对于IRTypeContext唯一化的类型，指针等价 ⟹ 结构等价。
  /// 对于非唯一化类型（如IRStructType按名查找），需逐字段比较。
  virtual bool equals(const IRType* Other) const = 0;

  virtual std::string toString() const = 0;

  /// 获取类型在目标平台上的大小（位）。
  /// 精确计算规则见2.1.3.3节。
  virtual uint64_t getSizeInBits(const TargetLayout& Layout) const = 0;

  /// 获取类型对齐（位）
  virtual uint64_t getAlignInBits(const TargetLayout& Layout) const = 0;

  /// 类型谓词
  bool isVoid() const { return KindVal == Void; }
  bool isBool() const { return KindVal == Bool; }
  bool isInteger() const { return KindVal == Integer; }
  bool isFloat() const { return KindVal == Float; }
  bool isPointer() const { return KindVal == Pointer; }
  bool isArray() const { return KindVal == Array; }
  bool isStruct() const { return KindVal == Struct; }
  bool isFunction() const { return KindVal == Function; }
  bool isVector() const { return KindVal == Vector; }
  bool isOpaque() const { return KindVal == Opaque; }

  /// 类方法支持（用于dyn_cast/isa模式）
  static bool classof(const IRType* T) { return true; }

protected:
  IRType(Kind K, DialectID D = DialectID::Core) : KindVal(K), DialectVal(D) {}
  Kind KindVal;
  DialectID DialectVal;
};

} // namespace blocktype::ir
```

```cpp
public:

    assert((N == 2 || N == 4 || N == 8 || N == 16 || N == 32 || N == 64)
           && "Vector NumElements must be power of 2");
  }
  unsigned getNumElements() const { return NumElements; }
  IRType* getElementType() const { return ElementType; }
  bool equals(const IRType* Other) const override;
  std::string toString() const override;
  uint64_t getSizeInBits(const TargetLayout& Layout) const override;
  uint64_t getAlignInBits(const TargetLayout& Layout) const override;
  static bool classof(const IRType* T) { return T->getKind() == Vector; }
};

class IROpaqueType : public IRType {
  std::string Name;
public:
  explicit IROpaqueType(StringRef N)
    : IRType(Opaque), Name(N.str()) {}
  StringRef getName() const { return Name; }
  bool equals(const IRType* Other) const override {
    return Other->isOpaque() && cast<IROpaqueType>(Other)->Name == Name;
  }
  std::string toString() const override { return "opaque %" + Name; }
  uint64_t getSizeInBits(const TargetLayout&) const override {
    assert(false && "Cannot get size of opaque type");
    return 0;
  }
  uint64_t getAlignInBits(const TargetLayout&) const override {
    assert(false && "Cannot get alignment of opaque type");
    return 0;
  }
  static bool classof(const IRType* T) { return T->getKind() == Opaque; }
};

} // namespace blocktype::ir
```

### 2.1.3.3 getSizeInBits 精确计算规则

| 类型 | 计算规则 | 示例（x86_64） |
|------|---------|---------------|
| `IRVoidType` | 固定返回0 | 0 |
| `IRIntegerType` | 返回`BitWidth` | i32→32, i64→64 |
| `IRFloatType` | 返回`BitWidth` | f32→32, f64→64, f80→80 |
| `IRPointerType` | 返回`Layout.getPointerSizeInBits()` | x86_64→64 |
| `IRArrayType` | `NumElements * ElementType::getSizeInBits()` + padding | [10 x i32]→320 |
| `IRStructType` | 非packed：按ABI对齐计算字段偏移，总和含尾部padding；packed：紧密排列 | {i8, i32}→64（非packed，含padding） |
| `IRFunctionType` | 固定返回0 | 0 |
| `IRVectorType` | `NumElements * ElementType::getSizeInBits()` | <4 x float>→128 |
| `IROpaqueType` | 触发assert（不可调用） | N/A |

**IRStructType 布局计算算法**：
1. 遍历字段，对每个字段计算对齐后的偏移：`offset = alignTo(currentOffset, fieldAlign)`
2. packed模式跳过对齐，偏移直接累加
3. 非packed模式尾部padding：`totalSize = alignTo(lastFieldEnd, structAlign)`
4. 结果缓存到`FieldOffsets`和`IsLayoutComputed`，避免重复计算

```cpp
namespace blocktype::ir {

class IRTypeContext {
  // 唯一化缓存
  DenseMap<unsigned, unique_ptr<IRIntegerType>> IntTypes;
  DenseMap<unsigned, unique_ptr<IRFloatType>> FloatTypes;
  FoldingSet<IRPointerType> PointerTypes;

  IRFloatType* getFloat80Ty()  { return getFloatType(80); }   // x86 extended
  IRFloatType* getFloat128Ty() { return getFloatType(128); }  // IEEE quad

  // === 派生类型工厂 ===

  /// 获取指针类型（唯一化：PointeeType+AddressSpace组合）
  IRPointerType* getPointerType(IRType* Pointee, unsigned AddressSpace = 0);

  /// 获取数组类型（唯一化：NumElements+ElementType组合）
  IRArrayType* getArrayType(IRType* Element, uint64_t Count);

  /// 获取向量类型（唯一化：NumElements+ElementType组合）
  IRVectorType* getVectorType(IRType* Element, unsigned Count);

  /// 获取命名结构体类型（按名唯一化）
  /// 如果名称已存在且字段不匹配，返回已有的（需调用setBody更新）
  IRStructType* getStructType(StringRef Name, SmallVector<IRType*, 16> Elems,
                               bool Packed = false);

  /// 创建匿名结构体（唯一化：按字段内容FoldingSet）
  IRStructType* getAnonStructType(SmallVector<IRType*, 16> Elems,
                                   bool Packed = false);

  /// 更新结构体体（用于前向声明→定义）
  bool setStructBody(IRStructType* S, SmallVector<IRType*, 16> Elems,
                      bool Packed = false);

  /// 获取函数类型（唯一化：ReturnType+ParamTypes+IsVarArg组合）
  IRFunctionType* getFunctionType(IRType* Ret, SmallVector<IRType*, 8> Params,
                                   bool VarArg = false);

  /// 获取不透明类型（按名唯一化）
  IROpaqueType* getOpaqueType(StringRef Name);

  // === 查询接口 ===

  /// 按名查找结构体
  IRStructType* getStructTypeByName(StringRef Name) const;

  /// 按名查找不透明类型
  IROpaqueType* getOpaqueTypeByName(StringRef Name) const;

  // === 统计 ===

  unsigned getNumTypesCreated() const { return NumTypesCreated; }
  size_t getMemoryUsage() const;
};

} // namespace blocktype::ir
```

**唯一化语义**：通过IRTypeContext获取的类型，相同参数返回相同指针。因此对于IRTypeContext产出的类型，`equals()`等价于指针比较。

```cpp
enum class FunctionAttr : uint32_t {
  NoReturn       = 1 << 0,   /// 不返回
  NoThrow        = 1 << 1,   /// 不抛异常
  ReadOnly       = 1 << 2,   /// 只读
  WriteOnly      = 1 << 3,   /// 只写
```

## 2.1.11 IRContext — IR 层内存管理

```cpp
namespace blocktype::ir {

class IRContext {
  /// IR 节点分配器（与 ASTContext::Allocator 同类型）
  llvm::BumpPtrAllocator Allocator;

  /// 非平凡资源清理回调（遵循 Clang/ASTContext 模式）
  std::vector<std::function<void()>> Cleanups;

  /// 类型唯一化上下文（拥有所有 IRType 实例）
  IRTypeContext TypeCtx;

  /// 线程安全策略
  IRThreadingMode ThreadingMode = IRThreadingMode::SingleThread;

public:
  /// 分配 IR 节点内存
  template <typename T, typename... Args>
  T *create(Args &&...args) {
    void *Mem = Allocator.Allocate(sizeof(T), alignof(T));
    T *Node = new (Mem) T(std::forward<Args>(args)...);
    return Node;
  }

  /// 注册非平凡资源清理
  void addCleanup(std::function<void()> Callback) {
    Cleanups.push_back(std::move(Callback));
  }

  /// 保存字符串到 IR 池
  llvm::StringRef saveString(llvm::StringRef Str);

  IRTypeContext &getTypeContext() { return TypeCtx; }

  /// 统计
  size_t getMemoryUsage() const { return Allocator.getTotalMemory(); }

  /// 线程安全接口
  void setThreadingMode(IRThreadingMode Mode) { ThreadingMode = Mode; }
  IRThreadingMode getThreadingMode() const { return ThreadingMode; }

  /// 标记 IRModule 为不可变（SharedReadOnly 模式下使用）
  void sealModule(IRModule &M);

  ~IRContext() {
    // 反向执行清理回调
    for (auto It = Cleanups.rbegin(); It != Cleanups.rend(); ++It)
      (*It)();
    // BumpPtrAllocator 自动释放所有内存
  }
};

} // namespace blocktype::ir
```

**与现有代码的对接**：
- `CompilerInstance` 新增 `unique_ptr<ir::IRContext> IRCtx` 成员
- `ASTToIRConverter` 持有 `ir::IRContext&` 引用，通过 `IRCtx.create<T>()` 分配 IR 节点
- `IRToLLVMConverter` 持有 `const ir::IRModule&` 引用，只读遍历

**性能预期**：
- 分配速度：BumpPtrAllocator O(1)，比 `new` 快 5-10x
- 内存开销：比 `unique_ptr` 逐个分配少 10-20%（无每对象元数据开销）
- 释放：一次性 `~IRContext()` 批量释放，无逐对象析构

## 2.1.12 IR 线程安全设计

```cpp
namespace blocktype::ir {

/// IRContext 线程安全策略
enum class IRThreadingMode {
  SingleThread,     /// 默认：无锁，最高性能
  MultiInstance,    /// 多实例：每个 CompilerInstance 拥有独立 IRContext
  SharedReadOnly    /// 共享只读：IRModule 可被多线程只读遍历
};

} // namespace blocktype::ir
```

**渐进式路线**：

| 阶段 | 模式 | 说明 |
|------|------|------|
| Phase A-E | `SingleThread` | 单线程编译，无锁开销 |
| 未来扩展1 | `MultiInstance` | 多文件并行编译，每线程独立 `CompilerInstance` + `IRContext` |
| 未来扩展2 | `SharedReadOnly` | IR 构建完成后 seal，多后端并行消费（如同时生成 x86_64 和 AArch64） |

**与 AI 模块对比**：AI 模块使用 `std::mutex` + `std::lock_guard` 保护共享状态。IR 层选择**不**在初期引入 mutex，因为：
- 编译管线是 CPU 密集型，锁竞争会显著影响性能
- 多实例模式通过数据隔离（每线程独立 IRContext）避免共享
- SharedReadOnly 模式通过 immutability 而非锁保证安全

**错误恢复策略**：

| 阶段 | 错误类型 | 恢复策略 |
|------|---------|---------|
| AST → IR 转换 | 类型映射失败 | 发出 Diag，用 `IROpaqueType` 占位，继续转换 |
| AST → IR 转换 | 语句转换失败 | 发出 Diag，跳过该语句，继续下一语句 |
| AST → IR 转换 | 表达式转换失败 | 发出 Diag，用 `IRConstantUndef` 占位，继续 |
| IR 验证 | 类型完整性 | 收集所有违规，一次性报告 |
| IR 验证 | SSA 性质 | 收集所有违规，一次性报告 |
| IR → LLVM 转换 | 不支持的 IR 特性 | 发出 Diag，中止该函数转换，继续下一函数 |
| IR → LLVM 转换 | 后端能力不足 | 检查 `BackendCapability`，提前报告不支持的特性 |

**与 SFINAE 机制的关系**：IR 转换层不需要 SFINAE 抑制（SFINAE 是模板推导语义，在 Sema 层已处理）。IR 转换的错误都是硬错误，直接通过 `DiagnosticsEngine` 报告。

## 2.1.14 IR 版本兼容性

```cpp
namespace blocktype::ir {

/// IR 格式版本（独立于编译器版本，允许 IR 格式独立演进）
struct IRFormatVersion {
  uint16_t Major;  /// 主版本：不兼容变更
  uint16_t Minor;  /// 次版本：新增特性，旧读取器可忽略
  uint16_t Patch;  /// 补丁版本：Bug 修复，完全兼容

  static constexpr IRFormatVersion Current() { return {1, 0, 0}; }

  bool isCompatibleWith(IRFormatVersion ReaderVersion) const {
    if (Major != ReaderVersion.Major) return false;
    if (ReaderVersion.Minor < Minor) return false;
    return true;
  }

  std::string toString() const {
    return std::to_string(Major) + "." + std::to_string(Minor) + "." + std::to_string(Patch);
  }
};

/// IR 二进制文件头
struct IRFileHeader {
  char Magic[4] = {'B', 'T', 'I', 'R'};  /// 魔数 "BTIR"
  IRFormatVersion Version = IRFormatVersion::Current();
  uint32_t Flags = 0;                      /// 保留标志位
  uint32_t ModuleOffset = 0;               /// 模块数据偏移
  uint32_t StringTableOffset = 0;          /// 字符串表偏移
  uint32_t StringTableSize = 0;            /// 字符串表大小
};

} // namespace blocktype::ir
```

**版本兼容策略**：

| 场景 | 处理 |
|------|------|
| 读取器版本 == 文件版本 | 完全兼容 |
| 读取器 Major == 文件 Major，读取器 Minor >= 文件 Minor | 向后兼容（忽略未知 Minor 特性） |
| 读取器 Major != 文件 Major | 拒绝加载，报告版本不兼容错误 |
| 文件有未知 Flags | 忽略未知标志位（向前兼容） |

**与 BMI 格式的对齐**：IR 二进制格式与 BMI 格式独立但设计对齐：
- 两者都使用 4 字节魔数 + 版本号 + 标志位
- IR 格式用于 IR 序列化（`--emit-btir-bitcode`/`--load-btir`）
- BMI 格式用于 C++20 模块接口序列化
- 未来可考虑统一文件头格式

### 2.1.19.3 DialectLoweringPass 完整降级规则表

```cpp
namespace blocktype::ir::dialect {

/// Dialect降级Pass：将高级Dialect指令降级为低级Dialect指令
class DialectLoweringPass : public Pass {
public:
  StringRef getName() const override { return "dialect-lowering"; }
  bool run(IRModule& M) override;

private:
  /// 降级规则：输入模式 → 输出模式 + 条件
  struct LoweringRule {
    DialectID SourceDialect;   /// 源Dialect
    Opcode SourceOpcode;       /// 源操作码
    DialectID TargetDialect;   /// 目标Dialect（必须为Core）
    /// 降级函数：将一条指令替换为多条Core指令
    /// 返回true表示成功降级
    std::function<bool(IRInstruction&, IRBuilder&)> Lower;
  };

  /// 降级规则表
  static const SmallVector<LoweringRule, 16> LoweringRules;
};

} // namespace blocktype::ir::dialect
```

**降级规则表**：

| # | 源Dialect | 源Opcode | 目标Dialect | 降级行为 | 条件 |
|---|----------|---------|------------|---------|------|
| 1 | bt_cpp | Invoke | bt_core | `invoke @f() to label %normal unwind label %unwind` → `call @f()` + landingpad布局 | 后端不支持异常处理 |
| 2 | bt_cpp | Resume | bt_core | `resume {ptr, i32} %val` → `unreachable`（Cranelift后端） | 后端不支持异常恢复 |
| 3 | bt_cpp | dynamic_cast | bt_core | `dynamic_cast<T>(v)` → `call @__dynamic_cast(v, &typeinfo_T, ...)` | 所有后端（dynamic_cast始终降级为函数调用） |
| 4 | bt_cpp | vtable.dispatch | bt_core | `vtable.dispatch(obj, idx)` → `load vptr` + `gep idx` + `indirect_call` | 所有后端（虚函数分派降级为间接调用） |
| 5 | bt_cpp | RTTI.typeid | bt_core | `typeid(expr)` → `call @__typeid(expr)` | 所有后端 |
| 6 | bt_target | target.intrinsic | bt_core | 目标特定intrinsic → 目标特定函数调用 | 后端不直接支持该intrinsic |
| 7 | bt_meta | meta.inline.always | bt_core | 附加函数属性`AlwaysInline` | 无条件（元数据→属性） |
| 8 | bt_meta | meta.inline.never | bt_core | 附加函数属性`NoInline` | 无条件 |
| 9 | bt_meta | meta.hot | bt_core | 附加函数属性`Hot` | 无条件 |
| 10 | bt_meta | meta.cold | bt_core | 附加函数属性`Cold` | 无条件 |

**Dialect 分层映射**：

| Dialect | 包含内容 | 消费者 |
|---------|---------|--------|
| `bt_core` | Add/Sub/Mul/Load/Store/Br/Call/Ret/ICmp/FCmp/Phi/Select/转换指令 | 所有后端 |
| `bt_cpp` | Invoke/Resume/LandingPad/DbgDeclare/DbgValue + C++语义元数据 | LLVM后端(完整), Cranelift后端(降级) |
| `bt_target` | 目标特定intrinsics(如x86 rdtsc, AArch64 hint) | 特定后端 |
| `bt_debug` | DICompileUnit/DIType/DISubprogram/DILocation | 支持调试的后端 |
| `bt_meta` | 优化提示(inline always/never, 热点标记) | 可选消费 |

**Dialect Lowering 管线**：

```
前端产出: bt_core + bt_cpp + bt_debug + bt_meta
    ↓ DialectLoweringPass (bt_cpp → bt_core)
    ↓   invoke → call + landingpad布局
    ↓   dynamic_cast → __dynamic_cast函数调用
    ↓   vtable → 全局常量 + 间接调用
    ↓
IR Pipeline: bt_core + bt_debug + bt_meta
    ↓ BackendCapability检查
    ↓   Cranelift不支持bt_cpp → 先Lowering再消费
    ↓   LLVM直接消费所有Dialect
    ↓
后端消费: bt_core + (bt_cpp if supported) + bt_debug + bt_meta
```

**与现有方案的兼容性**：
- DialectID 作为 `IRInstruction`/`IRType` 的附加字段，不改变现有接口
- 现有指令默认属于 `bt_core` Dialect
- `DialectLoweringPass` 是新增IR Pass，不影响现有Pass
- `BackendCapability` 扩展 `DialectCapability`，向后兼容

**行业对标**：
- MLIR Dialect：BTIR的Dialect是MLIR Dialect的轻量化实现
- GCC GIMPLE：GIMPLE也有high/low层级概念
- Swift SIL：SIL分为raw SIL和canonical SIL两层

```
tryMarkGreen(node):
  for each dependency of node:
    if dependency is Green: continue
    if dependency is Red: return false  // 输入变了，无法标绿
    if dependency is Unknown:
      if !tryMarkGreen(dependency): return false
      // 重新执行依赖查询，对比结果
      new_result = recompute(dependency)
      if new_result.fingerprint == old_fingerprint:
        markGreen(dependency)  // 假阳性优化！
      else:
        markRed(dependency)
        return false
  markGreen(node)
  return true
```

### 2.1.21.2 PhaseGuard 精确行为

```cpp
namespace blocktype::telemetry {

class TelemetryCollector {
  bool Enabled = false;
  SmallVector<CompilationEvent, 64> Events;
  uint64_t CompilationStartNs;

public:
  void enable() { Enabled = true; }
  bool isEnabled() const { return Enabled; }

  /// RAII阶段守卫
  class PhaseGuard {
    TelemetryCollector& Collector;
    CompilationPhase Phase;
    std::string Detail;
    uint64_t StartNs;       // 构造时记录的起始时间（clock_gettime CLOCK_MONOTONIC）
    size_t MemoryBefore;    // 构造时的内存使用量
  public:
    /// 构造函数：记录起始时间和内存
    PhaseGuard(TelemetryCollector& C, CompilationPhase P, StringRef D)
      : Collector(C), Phase(P), Detail(D.str()),
        StartNs(getCurrentTimeNs()),
        MemoryBefore(getCurrentMemoryUsage()) {}

    /// 析构函数：构造CompilationEvent并推入Events
    /// - EndNs = getCurrentTimeNs()
    /// - MemoryAfter = getCurrentMemoryUsage()
    /// - Success = true（默认；异常路径可通过markFailed()设置）
    /// - 如果!Collector.Enabled，不做任何操作
    ~PhaseGuard() {
      if (Collector.Enabled) {
        CompilationEvent E;
        E.Phase = Phase;
        E.Detail = Detail;
        E.StartNs = StartNs;
        E.EndNs = getCurrentTimeNs();
        E.MemoryBefore = MemoryBefore;
        E.MemoryAfter = getCurrentMemoryUsage();
        E.Success = !Failed;
        Collector.Events.push_back(std::move(E));
      }
    }

    /// 标记失败（在析构前调用）
    void markFailed() { Failed = true; }

  private:
    bool Failed = false;
    static uint64_t getCurrentTimeNs();    // clock_gettime(CLOCK_MONOTONIC)
    static size_t getCurrentMemoryUsage(); // /proc/self/statm or mallinfo()
  };

  PhaseGuard scopePhase(CompilationPhase P, StringRef Detail = "") {
    return PhaseGuard(*this, P, Detail);
  }

  // ... 其余接口不变 ...
};

} // namespace blocktype::telemetry
```

### IR 完整性验证

```cpp
namespace blocktype::ir::security {

/// IR完整性校验和
/// 哈希算法选择：BLAKE3
/// 理由：(1) 比CityHash64更安全（抗碰撞）(2) 比SHA-256更快（SIMD优化）
/// (3) 单文件头文件实现，无外部依赖
struct IRIntegrityChecksum {
  uint64_t TypeSystemHash;     /// 类型系统哈希（BLAKE3摘要前64位）
  uint64_t InstructionHash;    /// 指令序列哈希
  uint64_t ConstantHash;       /// 常量池哈希
  uint64_t GlobalHash;         /// 全局变量哈希
  uint64_t DebugInfoHash;      /// 调试信息哈希
  uint64_t CombinedHash;       /// 综合哈希（所有分量拼接后BLAKE3）

  static IRIntegrityChecksum compute(const IRModule& M);
  bool verify(const IRModule& M) const;
};

/// 密钥类型定义（Ed25519签名方案）
/// 选择理由：(1) 确定性签名（无随机数）(2) 短密钥+短签名 (3) libsodium成熟实现
using PrivateKey = std::array<uint8_t, 32>;  /// Ed25519私钥（32字节）
using PublicKey  = std::array<uint8_t, 32>;  /// Ed25519公钥（32字节）
using Signature  = std::array<uint8_t, 64>;  /// Ed25519签名（64字节）

/// IR签名（防止篡改）
class IRSigner {
public:
  /// 对IR模块签名
  /// 签名内容：BLAKE3(IRModule序列化字节)
  /// 签名算法：Ed25519
  static Signature sign(const IRModule& M, const PrivateKey& Key);

  /// 验证IR模块签名
  static bool verify(const IRModule& M, const Signature& Sig,
                     const PublicKey& Key);
};

} // namespace blocktype::ir::security
```

## 2.1.23 编译器缓存架构

> 引入多层编译缓存（参考 sccache/ccache 架构），加速重复编译。

```cpp
namespace blocktype::cache {

/// 缓存键：编译输入的哈希摘要
/// 哈希算法：BLAKE3（与IRIntegrityChecksum一致）
/// compute()流程：
/// 1. SourceHash = BLAKE3(source_file_bytes)
/// 2. OptionsHash = BLAKE3(serialized_compiler_options)
/// 3. VersionHash = BLAKE3(compiler_version_string + IRFormatVersion)
/// 4. TargetTripleHash = BLAKE3(target_triple_string)
/// 5. DependencyHash = BLAKE3(concatenated_dependency_hashes)
/// 6. CombinedHash = BLAKE3(SourceHash + OptionsHash + VersionHash + TargetTripleHash + DependencyHash)
struct CacheKey {
  uint64_t SourceHash;
  uint64_t OptionsHash;
  uint64_t VersionHash;
  uint64_t TargetTripleHash;
  uint64_t DependencyHash;
  uint64_t CombinedHash;

  static CacheKey compute(StringRef Source, const CompilerOptions& Opts);

  /// 用于磁盘文件命名
  std::string toHex() const;  // CombinedHash的十六进制表示
};

/// 缓存条目
struct CacheEntry {
  CacheKey Key;
  std::vector<uint8_t> IRData;          /// 序列化的IR（BTIR二进制格式）
  std::vector<uint8_t> ObjectData;      /// 编译的目标文件
  ir::IRFormatVersion IRVersion;        /// IR格式版本
  uint64_t Timestamp;                    /// 缓存时间戳（epoch秒）
  size_t IRSize;                         /// IR大小
  size_t ObjectSize;                     /// 目标文件大小
};

/// 本地磁盘缓存文件命名规则：
/// 目录结构：<CacheDir>/<first2hex>/<rest38hex>/
///   - IR缓存：<CacheDir>/ab/cdef0123456789.../ir.btir
///   - 目标缓存：<CacheDir>/ab/cdef0123456789.../obj.o
///   - 元数据：<CacheDir>/ab/cdef0123456789.../meta.json
/// 其中ab是CombinedHash十六进制的前2字符，cdef...是剩余38字符
/// 这种两级目录结构避免单目录过多文件（参考ccache设计）

/// 缓存条目序列化格式（meta.json）：
/// {
///   "sourceHash": "0x...",
///   "optionsHash": "0x...",
///   "versionHash": "0x...",
///   "targetTripleHash": "0x...",
///   "dependencyHash": "0x...",
///   "combinedHash": "0x...",
///   "irVersion": "1.0.0",
///   "timestamp": 1700000000,
///   "irSize": 1024,
///   "objectSize": 2048
/// }

/// 缓存存储接口（可插拔）
class CacheStorage {
public:
  virtual ~CacheStorage() = default;
  virtual Optional<CacheEntry> lookup(const CacheKey& Key) = 0;
  virtual bool store(const CacheKey& Key, const CacheEntry& Entry) = 0;
  struct Stats { size_t Hits; size_t Misses; size_t Evictions; size_t TotalSize; };
  virtual Stats getStats() const = 0;
};

/// 本地磁盘缓存
class LocalDiskCache : public CacheStorage {
  std::string CacheDir;
  size_t MaxSize;  /// 最大缓存大小（字节），默认10GB
public:

  Optional<CacheEntry> lookup(const CacheKey& Key) override;
  bool store(const CacheKey& Key, const CacheEntry& Entry) override;
  Stats getStats() const override;

  /// LRU淘汰：当TotalSize > MaxSize时，删除最旧的缓存条目
  void evictIfNeeded();
};

/// 远程缓存接口（S3/GCS/Azure，远期实现）
class RemoteCache : public CacheStorage {
  std::string Endpoint;
  std::string Bucket;
public:

class CompilationCacheManager {

} // namespace blocktype::cache
```

### 2.1.24.2 FFIOpcode 与主 Opcode 的关系

FFIOpcode已集成到主Opcode枚举中（见§2.1.5.2），占用160-175范围：
- `FFICall = 160`：外部函数调用（指定调用约定），语义与Call类似但附加CallingConvention
- `FFICheck = 161`：运行时类型检查（动态语言互操作），产出i1结果
- `FFICoerce = 162`：类型强制转换（跨语言类型映射），语义类似BitCast但允许不安全转换
- `FFIUnwind = 163`：外部异常边界（C++异常 ↔ 外部错误码），产出i32错误码

FFI指令均属于`bt_core` Dialect，因为FFI是通用机制而非C++特有。

## 2.1.25 IR 调试信息升级

> 在 IR 层建立调试元数据体系，支持 DWARF5 标准，升级现有 §2.1.10 的基础调试信息设计。

### 与现有§2.1.10 DebugMetadata的关系

§2.1.10的`DebugMetadata`/`DICompileUnit`/`DIType`/`DISubprogram`/`DILocation`是**基础调试信息**，位于`blocktype::ir`命名空间。
§2.1.25的`IRDebugMetadata`/`IRInstructionDebugInfo`/`DebugInfoEmitter`是**调试信息升级**，位于`blocktype::ir::debug`命名空间。

**升级策略：共存+渐进替换**
- Phase A-D：§2.1.10基础调试信息继续使用（后端映射到`llvm::DIDescriptor`）
- Phase A-D：§2.1.25升级调试信息**并行存在**，`IRInstruction::DbgInfo`字段可选启用
- Phase E+：§2.1.25完全替代§2.1.10，§2.1.10标记为deprecated
- 映射关系：`ir::debug::IRDebugMetadata` → `ir::DebugMetadata`（适配层）

```cpp
namespace blocktype::ir::debug {

/// IR指令调试标注（每条IR指令可关联源位置）
class IRInstructionDebugInfo {
  IRDebugMetadata::SourceLocation Loc;     // 40字节：StringRef(16) + 4*uint32_t(16) + padding(8)
  Optional<IRDebugMetadata::DISubprogram*> Subprogram;  // 16字节：指针+has_value
  bool IsArtificial;                       // 1字节
  bool IsInlined;                          // 1字节
  Optional<IRDebugMetadata::SourceLocation> InlinedAt;  // 48字节：Optional<SourceLocation>
  // 总计约106字节（含padding约112字节）
};

/// Optional开销分析：
/// - Optional<T> = sizeof(T) + 1字节(has_value标志) + padding
/// - Optional<DISubprogram*> = 16字节（8字节指针 + 1字节flag + 7字节padding）
/// - Optional<SourceLocation> = 48字节（40字节内容 + 1字节flag + 7字节padding）
/// - IRInstructionDebugInfo总计约112字节
///
/// 对IRInstruction内存影响：
/// - 无调试信息：Optional<IRInstructionDebugInfo> = 1字节（仅has_value=false）
/// - 有调试信息：Optional<IRInstructionDebugInfo> = 113字节（112+1）
/// - 内存开销：平均每条指令增加约1-113字节，取决于是否启用-g
///
/// 优化策略：
/// - Release构建不存储DbgInfo（Optional始终为None，开销仅1字节）
/// - Debug构建启用DbgInfo时，可考虑将Subprogram和InlinedAt
///   移到SideTable以减少每指令开销（远期优化）

} // namespace blocktype::ir::debug
```
