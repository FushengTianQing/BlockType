## 附录 A：现有代码耦合分析

### A.1 LLVM 直接依赖统计

| 文件 | LLVM类型引用 | 耦合程度 |
|------|-------------|---------|
| CodeGenModule.h | llvm::Module, llvm::LLVMContext, llvm::DenseMap, llvm::GlobalValue | 高 |
| CodeGenFunction.h | llvm::IRBuilder, llvm::Function, llvm::Value, llvm::AllocaInst | 高 |
| CodeGenTypes.h | llvm::Type, llvm::FunctionType, llvm::StructType | 高 |
| CodeGenConstant.h | llvm::Constant, llvm::APSInt, llvm::APFloat | 高 |
| CGCXX.h | llvm::GlobalVariable, llvm::Function, llvm::Value | 高 |
| CGDebugInfo.h | llvm::DIBuilder, llvm::DIType, llvm::DISubprogram | 高 |
| Mangler.h | llvm::StringRef, llvm::SmallString (仅ADT) | 低 |
| TargetInfo.h | llvm::DataLayout, llvm::StringRef | 中 |
| ABIInfo.h | 通过CodeGenTypes间接依赖 | 中 |
| CompilerInstance.h | llvm::LLVMContext | 中 |
| Basic/LLVM.h | llvm ADT类型别名 | 低 |

### A.2 关键耦合点

1. **CodeGenModule→llvm::Module**：模块级引擎直接持有LLVM Module
2. **CodeGenFunction→llvm::IRBuilder**：函数级引擎直接使用LLVM IRBuilder
3. **CodeGenTypes→llvm::Type***：类型映射直接产出LLVM类型
4. **CompilerInstance→llvm::LLVMContext**：编译实例直接持有LLVM上下文
5. **CGDebugInfo→llvm::DIBuilder**：调试信息直接使用LLVM DIBuilder

### A.3 已有抽象可复用

1. **TargetInfo抽象基类+工厂方法**：设计模式可直接复用到FrontendBase/BackendBase
2. **ABIInfo抽象基类+工厂方法**：同上
3. **Basic/LLVM.h ADT类型别名**：IR层可复用这些ADT类型

## 附录 G：代码库现状摘要（影响补充设计的关键事实）

> 扫描日期：2026-04-24 | 扫描者：architect-v3

| 维度 | 现状 | 影响补充设计 |
|------|------|-------------|
| AST 内存管理 | `ASTContext` 使用 `llvm::BumpPtrAllocator` | IR 层沿用此模式 → IRContext |
| CodeGen 内存管理 | `CodeGenModule` 子组件全部 `unique_ptr` | IR Module 层沿用 `unique_ptr` 组合 |
| 编译管线线程安全 | 核心编译路径**无**任何 mutex/lock/atomic | IR 层初期无需线程安全，但需预留接口 |
| AI 模块线程安全 | `AIOrchestrator`/`ConnectionPool`/`ConcurrentRequest` 完整 mutex 保护 | 可参考 AI 模块的并发模式 |
| 错误恢复 | `DiagnosticsEngine` 有 SFINAE 抑制（`pushSuppression`/`popSuppression`），`Parser` 有 skipUntil 恢复，`Sema` 有 `ExprResult`/`StmtResult` | IR 转换错误应复用 `DiagnosticsEngine` |
| 版本机制 | `Version.h` 有 `BLOCKTYPE_VERSION_STRING "0.1.0"`，`BMIFormat.h` 有 `BMIFlags` 枚举 | IR 版本可复用版本号体系 |
| Pass 框架 | **无** IR 级别 Pass，优化完全委托 `llvm::PassBuilder` | IR Pass 框架需从零建设 |
| 增量编译 | **无** 增量编译基础设施，`ModuleManager` 有模块缓存但非增量 | 增量编译为远期目标 |
| 后端能力查询 | `TargetInfo` 有 `isX86_64()`/`isAArch64()`/`isLinux()` 等查询 | 可扩展为 IR 特性子集声明 |
| 测试框架 | GTest + lit，61 个 unit test，33 个 lit test | 契约测试可融入现有框架 |

## 附录 E：IR 文本格式完整示例

```
; BlockType IR (BTIR) - 文本表示
; Dialect标注语法：指令和类型可标注所属Dialect
; 格式：<instruction> !dialect!<dialect_name>
; 默认Dialect为bt_core，无需标注；非Core Dialect必须标注

target triple = "x86_64-unknown-linux-gnu"
target datalayout = "e-m:e-p270:32:32-p271:32:32-p272:64:64-i64:64-f80:128-n8:16:32:64-S128"

; 全局变量
@global_counter = global i32 0
@pi = constant double 3.14159265358979

; 外部函数声明
declare i32 @printf(i8*, ...)
declare void @exit(i32)

; 函数定义（bt_core Dialect，默认无需标注）
define i32 @add(i32 %a, i32 %b) {
entry:
  %result = add i32 %a, %b
  ret i32 %result
}

define i32 @main() {
entry:
  %0 = call i32 @add(i32 1, i32 2)
  store i32 %0, i32* @global_counter
  ret i32 0
}

; C++ 类示例（bt_cpp Dialect标注）
%struct.Point = type { double, double }

define void @Point.constructor(%struct.Point* %this, double %x, double %y) {
entry:
  %x.ptr = getelementptr %struct.Point, %struct.Point* %this, i32 0, i32 0
  store double %x, double* %x.ptr
  %y.ptr = getelementptr %struct.Point, %struct.Point* %this, i32 0, i32 1
  store double %y, double* %y.ptr
  ret void
}

; bt_cpp Dialect示例：异常处理
define i32 @try_catch_example() !dialect!bt_cpp {
entry:
  %result = invoke i32 @may_throw() to %normal unwind %cleanup !dialect!bt_cpp
  br %normal

normal:
  ret i32 %result

cleanup:
  %exc = landingpad %struct.ExceptionType !dialect!bt_cpp
  resume %struct.ExceptionType %exc
}

; bt_debug Dialect示例：调试信息标注
define i32 @debug_example() !dialect!bt_debug {
entry:
  %x = add i32 %a, %b !dbg !42 !dialect!bt_debug
  ret i32 %x
}

; Dialect标注语法总结：
; 1. 指令级标注：<instruction> !dialect!<dialect_name>
;    示例：add i32 %a, %b !dialect!bt_cpp
; 2. 函数级标注：define <type> @name(...) !dialect!<dialect_name> { ... }
;    示例：define i32 @foo() !dialect!bt_cpp { ... }
; 3. 类型级标注：<type> !dialect!<dialect_name>
;    示例：%struct.ExceptionType !dialect!bt_cpp
; 4. 默认省略：bt_core Dialect的指令/类型无需标注
;    示例：add i32 %a, %b（隐含!dialect!bt_core）
; 5. 多标注：指令可同时有调试和Dialect标注
;    示例：add i32 %a, %b !dbg !42 !dialect!bt_core
```
