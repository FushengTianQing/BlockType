## 使用示例

### 纯英文模式

```cpp
#include <iostream>

int main() {
    int counter = 0;
    if (counter > 0) {
        return counter;
    } else {
        return -counter;
    }
}
```

### 纯中文模式

```cpp
#包含 <iostream>

整数 主函数() {
    整数 计数器 = 0;
    如果 (计数器 > 0) {
        返回 计数器;
    } 否则 {
        返回 -计数器;
    }
}
```

### 混合模式

```cpp
#include <iostream>

整数 主函数() {
    int 计数器 = 0;
    如果 (counter > 0) {
        return 计数器;
    } else {
        返回 -counter;
    }
}
```

## 实现注意事项

### 1. 关键字识别

- Lexer 需要同时识别中英文关键字
- 使用统一的 Token 类型表示同一关键字的不同语言形式
- 例如：`if` 和 `如果` 都映射到 `tok::kw_if`

### 2. Token 表示

```cpp
enum class TokenKind {
  // 关键字
  kw_if,      // if 或 如果
  kw_else,    // else 或 否则
  kw_while,   // while 或 当
  // ...
};

class Token {
  TokenKind Kind;
  std::string Text;  // 原始文本（"if" 或 "如果"）
  SourceLocation Location;
};
```

### 3. 诊断信息

- 诊断信息应使用与源代码相同的语言
- 例如：如果源代码使用中文关键字，错误信息也应使用中文

### 4. 兼容性

- 生成的目标码与标准 C++ 完全兼容
- 中文关键字在目标码中转换为对应的英文形式

## 总结

BlockType 的关键字映射设计遵循以下原则：

1. **语义一致**：中英文关键字语义完全相同
2. **自然直观**：中文关键字选择符合中文表达习惯
3. **混用灵活**：允许在同一文件中混用中英文关键字
4. **兼容标准**：生成的目标码与标准 C++ 完全兼容
