# 开发-前中后-三阶段-核查要求

————————————一次写丰富开发plan

对Plan文件夹Phase4的开发文档进行优化

1.优化功能，提升完整性
已有的功能，进行优化。对缺失的功能进行补充，可查看 `docs/ARCHITECTURE.md` 中的模块职责划分
并参照 Clang 的对应实现做决策。

2.优化接口描述，保证完整性

每个模块产出的**接口（头文件中的类/函数声明）必须是完整的**，即使实现是空壳。

- 缺少的方法声明 → 立即补全
- 缺少的 enum 值 → 立即补全
- 缺少的 `classof` 覆盖 → 立即补全
- 缺少的构造函数/工厂方法 → 立即补全

理由：后续阶段的 AI 开发者看到完整接口时，会使用正确的方式；看到残缺接口时，会发明临时的替代方案。
