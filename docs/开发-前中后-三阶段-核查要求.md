2.优化接口描述，保证完整性
每个模块都应该有接口，如果文档没写，请补充。
每个模块产出的**接口（头文件中的类/函数声明）必须是完整的**，即使实现是空壳。

- 缺少的方法声明 → 立即补全
- 缺少的 enum 值 → 立即补全
- 缺少的 `classof` 覆盖 → 立即补全
- 缺少的构造函数/工厂方法 → 立即补全

理由：后续阶段的 AI 开发者看到完整接口时，会使用正确的方式；看到残缺接口时，会发明临时的替代方案。
