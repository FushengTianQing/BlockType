## P0 问题（必须修复）

### P0-1: TypeDeduction.cpp 自动类型推导实现严重不足

**文件**: `src/Sema/TypeDeduction.cpp`  
**问题描述**: 自动类型推导（`auto`/`decltype`/`decltype(auto)`）的实现非常简化：
1. `auto` 仅处理简单的初始化器类型推导，不支持 `auto&`、`auto&&`、`const auto&` 等引用和 cv 限定符组合
2. `decltype` 不处理 `decltype((x))`（双括号返回引用类型）的情况
3. `decltype(auto)` 未实现
4. 返回类型推导（`auto func() { return expr; }`）未实现
5. 结构化绑定的类型推导未实现  
**影响**: C++11/14/17/20/23 大量使用 auto 和 decltype 的代码无法正确编译。  
**修改建议**: 参考 Clang 的 `SemaType::DeduceAutoType()` 实现，完整支持所有 auto/decltype 推导场景。  
**预期完成时间**: 10 天

### P0-2: Overload.cpp 重载解析缺少排名机制

**文件**: `src/Sema/Overload.cpp`  
**问题描述**: 重载解析的实现缺少关键的排名（ranking）机制：
1. 没有实现隐式转换序列的排名（标准转换 > 用户定义转换 > 省略号转换）
2. 没有实现偏序排序（partial ordering）用于模板函数重载
3. 没有实现 tiebreaker 规则（非模板优先于模板、更特化优先等）
4. 当多个候选函数匹配时，没有歧义诊断  
**影响**: 任何非平凡的重载场景都会产生错误结果或静默选择错误的候选。  
**修改建议**: 参考 Clang 的 `OverloadCandidateSet::BestViableFunction()` 实现，添加完整的隐式转换序列排名和偏序排序。  
**预期完成时间**: 15 天

### P0-3: 模板实例化关键路径未打通

**文件**: `src/Sema/TemplateInstantiation.cpp`, `src/Sema/TemplateInstantiator.cpp`  
**问题描述**: 模板实例化的实现存在多个断裂点：
1. `TemplateInstantiator` 没有正确处理依赖类型的表达式替换
2. 模板参数推导与实例化之间的衔接不完整
3. 递归模板实例化没有深度限制，可能导致无限递归
4. 实例化后的类型替换没有 SFINAE 保护——替换失败直接报错而非从重载集中移除  
**影响**: 任何非平凡的模板代码（如 STL 容器、算法）都无法正确实例化。  
**修改建议**: 
1. 实现完整的 SFINAE 保护机制
2. 添加模板实例化深度限制（建议 1024 层）
3. 完善 `TemplateInstantiator` 的类型和表达式替换逻辑  
**预期完成时间**: 20 天

## P1 问题（高优先级）

### P1-1: 异常处理 Sema 分析严重不足

**文件**: `src/Sema/Sema.cpp` 中异常相关代码  
**问题描述**: 异常处理的 Sema 分析仅完成约 20%：
1. 没有 `throw` 表达式的类型检查（抛出类型必须可复制/可移动）
2. 没有 `catch` 子句的可达性分析
3. 没有 `noexcept` 规范的求值和违反检测
4. 没有 `noexcept(expr)` 运算符的求值
5. 没有异常规范与函数类型系统的集成  
**修改建议**: 参考 Clang 的异常处理 Sema 实现，至少完成：
1. throw 表达式类型检查
2. catch 子句匹配分析
3. noexcept 规范检查  
**预期完成时间**: 10 天

### P1-2: ConstraintSatisfaction.cpp 约束满足检查不完整

**文件**: `src/Sema/ConstraintSatisfaction.cpp`  
**问题描述**: C++20 概念的约束满足检查实现简化：
1. 没有实现约束的原子表达式归一化（normalization）
2. 没有实现约束的偏序（更严格的约束优先）
3. `requires` 子句的求值没有完整的子stitution 失败处理  
**修改建议**: 完善约束归一化和偏序排序。  
**预期完成时间**: 7 天

### P1-3: Sema.cpp 文件过大（3273 行）

**文件**: `src/Sema/Sema.cpp`  
**问题描述**: Sema.cpp 包含了过多职责：声明处理、表达式分析、语句分析、类型检查入口等。Clang 将这些分散到 SemaDecl.cpp、SemaExpr.cpp、SemaStmt.cpp、SemaType.cpp 等十几个文件中。  
**修改建议**: 按职责拆分 Sema.cpp：
- `SemaDecl.cpp` — 声明处理
- `SemaExpr.cpp` — 表达式分析
- `SemaStmt.cpp` — 语句分析
- `SemaType.cpp` — 类型检查入口
- `SemaOverload.cpp` — 重载解析
- `SemaTemplate.cpp` — 模板相关  
**预期完成时间**: 5 天

### P1-4: 缺少 SFINAE 保护

**文件**: 全 Sema 模块  
**问题描述**: 当前模板参数推导和替换失败时直接报错，而非从重载候选集中移除。这是 C++ 模板元编程的基础机制，缺少 SFINAE 会导致：
1. `std::enable_if` 无法工作
2. `std::void_t` 技巧无法使用
3. 任何基于 SFINAE 的条件编译失败  
**修改建议**: 在模板参数推导和约束检查中实现 SFINAE 上下文，替换失败时从候选集中移除而非报错。  
**预期完成时间**: 10 天

### P1-5: Lambda 表达式 Sema 分析不完整

**文件**: `src/Sema/Sema.cpp` 中 Lambda 相关代码  
**问题描述**: Lambda 的 Sema 分析缺少：
1. 捕获变量的可达性检查（是否 ODR 使用）
2. 初始化捕获的类型推导
3. 泛型 Lambda 的模板参数推导
4. Lambda 到函数指针的转换  
**修改建议**: 完善 Lambda 的 Sema 分析，至少覆盖 C++14 泛型 Lambda。  
**预期完成时间**: 7 天

## P2 问题（中优先级）

### P2-1: TypeCheck.cpp 缺少枚举类型检查

**文件**: `src/Sema/TypeCheck.cpp`  
**问题描述**: 类型检查没有处理枚举类型的底层类型推导、枚举值范围检查、scoped enum 的隐式转换限制等。  
**修改建议**: 添加枚举类型的完整类型检查。  
**预期完成时间**: 3 天

### P2-2: 缺少未定义行为检测

**文件**: 全 Sema 模块  
**问题描述**: 没有检测常见的未定义行为：
1. 有符号整数溢出
2. 空指针解引用
3. 越界数组访问
4. 使用未初始化的变量  
**修改建议**: 至少添加 `-Wuninitialized` 和 `-Wnull-dereference` 级别的警告。  
**预期完成时间**: 5 天

### P2-3: 隐式转换序列不完整

**文件**: `src/Sema/TypeCheck.cpp`  
**问题描述**: 隐式转换只处理了基本类型之间的转换，缺少：
1. 用户定义转换（构造函数和转换运算符）
2. 派生类到基类的指针/引用转换
3. 限定符转换（添加 const）
4. 数组到指针衰减  
**修改建议**: 完善隐式转换序列，至少覆盖标准转换。  
**预期完成时间**: 7 天

### P2-4: 缺少访问控制检查

**文件**: 全 Sema 模块  
**问题描述**: 没有实现 C++ 的访问控制（public/protected/private）。任何成员都可以从任何地方访问。  
**修改建议**: 在 Sema 层实现访问控制检查，至少对成员访问和基类访问进行检查。  
**预期完成时间**: 5 天
